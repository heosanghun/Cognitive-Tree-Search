## 1. 이 프로젝트가 하려는 것

### 한 줄 요약

> **작은 AI 모델(Gemma 4 E4B)이 수학·코딩·추론 문제를 더 잘 풀 수 있도록,
> "생각하는 방법"을 가르치는 새로운 기술(CTS)을 만들고, 정말 효과가 있는지 시험으로 증명하는 것.**

### 비유로 이해하기

학교 시험을 생각해 보세요:

- **보통 학생** = 문제를 보자마자 바로 답을 적습니다 → 이것이 **Greedy**
- **꼼꼼한 학생** = 같은 문제를 여러 번 풀어보고, 가장 많이 나온 답을 고릅니다 → 이것이 **SC@14**
- **천재 학생** = 머릿속으로 깊이 생각한 뒤 답을 적습니다 → 이것이 **Native Think**
- **전략적 학생** = 여러 풀이 경로를 나무(트리)처럼 그려보고, 가장 유망한 경로를 탐색합니다 → 이것이 **MCTS**
- **CTS 학생** = 위의 전략적 탐색을 하되, **메모리를 거의 안 쓰면서** 더 깊이 생각할 수 있습니다 → 이것이 **CTS (우리의 방법)**

이 프로젝트는 "CTS 학생"이 정말 다른 학생들보다 시험을 잘 보는지를 **5가지 시험**으로 검증합니다.

## 2. 벤치마크란 무엇인가?

**벤치마크(Benchmark)** = AI의 성적표를 만들기 위한 **표준화된 시험**

사람이 수능, 토익, 올림피아드 같은 시험으로 능력을 측정하듯이,
AI도 정해진 시험 문제를 풀게 하여 성능을 **객관적인 숫자**로 비교합니다.

### 왜 벤치마크가 필요한가?

| 이유 | 설명 |
|------|------|
| **객관적 비교** | "우리 AI가 좋다"는 말만으로는 증명이 안 됩니다. 같은 시험을 보게 해야 공정하게 비교할 수 있습니다 |
| **재현 가능** | 다른 연구자도 같은 시험지로 같은 실험을 반복할 수 있어야 합니다 |
| **다양한 능력 측정** | 수학만 잘하면 안 됩니다. 코딩, 추론, 응용력 등 다양한 능력을 봐야 합니다 |
| **논문 심사 기준** | 학술 논문(이 프로젝트는 NeurIPS 2026 학회 대상)에서 "우리 방법이 좋다"를 증명하려면 반드시 벤치마크 결과가 필요합니다 |

## 3. 5가지 벤치마크 상세 설명

### 3.1 MATH-500

| 항목 | 내용 |
|------|------|
| **정식 명칭** | MATH Benchmark (Hendrycks et al.) — 500문제 서브셋 |
| **"500"의 의미** | 전체 MATH 데이터셋(12,500문제)에서 **대표 500문제**를 추출한 것. "MATH-500"에서 500은 문제 수입니다 |
| **출처** | HuggingFace: `HuggingFaceH4/MATH-500` |
| **만든 곳** | UC Berkeley (Hendrycks 교수 연구팀) |
| **문제 유형** | 미국 고교~대학 수준 수학 (대수, 기하, 확률, 정수론, 미적분 등 7개 분야) |
| **난이도** | Level 1(쉬움) ~ Level 5(매우 어려움)까지 5단계. Level 5는 수학 올림피아드 수준 |
| **답 형식** | 정확한 수학적 답 (예: `\\frac{3}{4}`, `42`, `\\sqrt{2}`) |
| **왜 사용?** | AI의 **수학적 추론 능력**을 측정하는 가장 유명한 벤치마크 중 하나 |

**문제 예시 (Level 2, 대수):**

> 방정식 `2x + 5 = 13`을 만족하는 x의 값을 구하시오.
>
> 정답: `4`

**문제 예시 (Level 5, 정수론):**

> 1000 이하의 양의 정수 중 12와 18의 공배수이면서 24의 배수가 아닌 수의 개수를 구하시오.
>
> 정답: `\\frac{1000}{36} - \\frac{1000}{72}` → 계산 결과

### 3.2 GSM8K

| 항목 | 내용 |
|------|------|
| **정식 명칭** | Grade School Math 8K |
| **"8K"의 의미** | 약 **8,000개**의 문제로 구성 (학습용 7,473개 + 시험용 **1,319개**) |
| **시험 문제 수** | **1,319문제** (test split) |
| **출처** | HuggingFace: `gsm8k` (OpenAI가 제작) |
| **만든 곳** | OpenAI |
| **문제 유형** | 초등학교~중학교 수준의 **산술 문장제** (word problems) |
| **난이도** | 비교적 쉬움. 사칙연산 2~8단계로 풀 수 있는 실생활 문제 |
| **답 형식** | 정수 (예: `72`, `150`) — 풀이 마지막에 `#### 숫자` 형태로 정답 표기 |
| **왜 사용?** | 기본적인 **다단계 산술 추론** 능력을 측정. "쉬운 문제를 얼마나 안정적으로 풀 수 있는가"를 확인 |

**문제 예시:**

> 나탈리는 사과 42개를 가지고 있습니다. 그녀는 친구에게 사과의 절반을 나눠주었습니다.
> 나머지 사과 중 1/3을 동생에게 주었습니다.
> 나탈리에게 남은 사과는 몇 개입니까?
>
> 풀이: 42 ÷ 2 = 21 → 21 - (21 ÷ 3) = 21 - 7 = 14
> 정답: `#### 14`

### 3.3 AIME

| 항목 | 내용 |
|------|------|
| **정식 명칭** | American Invitational Mathematics Examination |
| **한국어** | 미국 수학 초대 시험 (미국 수학 올림피아드의 예선) |
| **시험 문제 수** | **약 90문제** (연도별 수집, 데이터셋에 따라 다름) |
| **출처** | HuggingFace: `AI-MO/aimo-validation-aime` 또는 `Maxwell-Jia/AIME_2024` |
| **만든 곳** | MAA (미국수학협회) — 실제 수학 올림피아드 대회 문제 |
| **문제 유형** | 고급 수학 — 조합론, 정수론, 기하학, 대수학 |
| **난이도** | **매우 어려움** — 미국 전역에서 상위 5% 학생만 응시하는 대회 문제. 한국의 KMO(한국수학올림피아드) 수준 |
| **답 형식** | 000~999 사이의 정수 |
| **왜 사용?** | AI의 **깊은 수학적 사고력**을 측정. 단순 암기나 패턴 매칭으로는 풀 수 없는 문제들 |

**문제 예시:**

> 양의 정수 n에 대해 n²을 1000으로 나눈 나머지가 1이 되는
> 1 ≤ n ≤ 1000인 n의 개수를 구하시오.
>
> 정답: 정수 (예: `10`)

**왜 중요한가?**

AIME는 실제 수학 올림피아드 문제이기 때문에, AI가 이 문제를 잘 풀 수 있다면
**"진짜 수학적 사고"** 를 할 수 있다는 강력한 증거가 됩니다.

### 3.4 ARC-AGI-Text

| 항목 | 내용 |
|------|------|
| **정식 명칭** | Abstraction and Reasoning Corpus (텍스트 변환 버전) |
| **시험 문제 수** | **약 200문제** (다운로드 설정에 따라 다름) |
| **출처** | HuggingFace: `barc0/200k_HEAVY_...` 또는 `allenai/ai2_arc` (ARC-Challenge) |
| **만든 곳** | François Chollet (Keras 창시자, Google 연구원) |
| **문제 유형** | **패턴 인식 + 추상적 추론** — 입력 패턴을 보고 규칙을 찾아 출력을 예측 |
| **난이도** | 어려움 — 사람은 쉽게 풀지만 AI에게는 매우 어려운 유형 |
| **답 형식** | 패턴/그리드 출력 (텍스트로 변환) |
| **왜 사용?** | AI의 **일반화 추론 능력(AGI에 가까운 능력)** 을 측정. 암기가 아닌 진짜 "이해"를 테스트 |

**문제 예시 (단순화):**

> 입력: `[[1, 0, 0], [0, 1, 0], [0, 0, 1]]`
> 출력: `[[0, 0, 1], [0, 1, 0], [1, 0, 0]]`
>
> (규칙: 행렬을 좌우 반전)

**왜 중요한가?**

ARC는 "AI가 정말 새로운 규칙을 스스로 발견할 수 있는가?"를 테스트합니다.
기존에 본 적 없는 패턴도 풀어야 하므로, **진정한 추론 능력**의 척도입니다.

## 4. 추론 전략(풀이 방법) 용어 설명

AI에게 문제를 풀게 할 때, **"어떤 방식으로 풀게 하느냐"** 에 따라 성능이 크게 달라집니다.
이것을 **추론 전략(Inference Strategy)** 이라고 합니다.

### 4.1 Greedy (그리디)

| 항목 | 내용 |
|------|------|
| **뜻** | "탐욕적" — 매 순간 가장 확률이 높은 단어를 하나씩 선택 |
| **비유** | 시험에서 **한 번만 풀고 바로 제출**하는 것 |
| **장점** | 가장 빠름 (계산 비용 최소) |
| **단점** | 한 번 틀린 방향으로 가면 돌아올 수 없음. 단순한 실수에도 취약 |
| **MACs** | 0.05 × 10¹⁴ (매우 적은 계산량) |

**동작 방식:**

```
문제 → AI가 한 글자씩 생성 → 완성된 답 제출
             (매번 가장 확률 높은 글자 선택)
```

### 4.2 Native Think (네이티브 씽크)

| 항목 | 내용 |
|------|------|
| **뜻** | Gemma 모델에 내장된 "생각하기" 모드 |
| **비유** | 시험에서 **연습장에 풀이과정을 쓴 뒤** 답을 적는 것 |
| **장점** | 모델이 스스로 추론 과정을 생성하므로 더 정확 |
| **단점** | 생각하는 만큼 시간(계산)이 더 걸림 |
| **MACs** | ≈ 0.80 × 10¹⁴ |

**동작 방식:**

```
문제 → <|think|> 생각 과정... </think> → 최종 답 생성
       (모델이 자체적으로 추론 과정을 텍스트로 생성)
```

**왜 비교하는가?**

CTS가 Native Think보다 더 효과적인 "생각하기" 방법임을 증명하기 위해서입니다.
같은 모델(Gemma 4 E4B)이 내장 기능으로 생각할 때 vs CTS로 생각할 때를 비교합니다.

### 4.3 SC@14 (셀프 컨시스턴시)

| 항목 | 내용 |
|------|------|
| **정식 명칭** | Self-Consistency with N=14 samples |
| **"SC"의 뜻** | Self-Consistency = 자기 일관성 |
| **"@14"의 뜻** | 14번 반복한다는 의미. 즉, **같은 문제를 14번 풀어본다** |
| **"T=0.7"의 뜻** | Temperature=0.7 — AI의 "창의성" 수준. 높을수록 다양한 답이 나옴 |
| **비유** | 시험 문제를 **14번 풀어보고, 가장 많이 나온 답을 최종 답으로 제출** (다수결) |
| **장점** | 여러 번 풀어서 다수결 하므로 안정적 |
| **단점** | 14배의 계산 비용 필요 → 매우 느림 |
| **MACs** | 1.0 × 10¹⁴ (Greedy의 20배) |

**동작 방식:**

```
문제 → [풀이1: 답=42] [풀이2: 답=42] [풀이3: 답=38] ... [풀이14: 답=42]
     → 다수결 → 최종 답: 42 (14번 중 11번 나왔으므로)
```

**왜 건너뛰었나?**

이전 실험에서 SC@14는 **실행하지 않기로** 결정했습니다. 이유:
- 14번 반복이므로 Greedy의 **14배 시간**이 걸림
- GSM8K 1,319문제 × 14회 = 약 112시간 이상 소요
- 논문의 SC@14 수치는 이미 확정되어 있으므로 재현 우선순위가 낮음

### 4.4 MCTS (몬테카를로 트리 서치)

| 항목 | 내용 |
|------|------|
| **정식 명칭** | Monte Carlo Tree Search |
| **뜻** | 여러 풀이 경로를 **나무(트리) 구조**로 탐색하는 알고리즘 |
| **비유** | 바둑에서 여러 수를 미리 둬보고, 가장 유리한 수를 찾는 것 (AlphaGo가 사용한 기술) |
| **문제점** | AI 언어 모델에 적용하면 **KV 캐시**가 트리 깊이에 비례하여 폭발적으로 증가 → **메모리 초과(OOM)** |

**두 가지 변형:**

| 변형 | 설명 | 한계 |
|------|------|------|
| **MCTS (Constrained)** | 메모리 제한 내에서만 탐색. 깊이 15에서 OOM | MACs 0.2에서 중단 — 충분히 탐색 못함 |
| **MCTS (Early Stop)** | OOM 전에 미리 멈춤. 더 깊이 탐색 가능 | 여전히 깊이 35에서 한계 |

**OOM이란?**

OOM = Out Of Memory = 메모리 부족.
GPU의 VRAM(비디오 메모리) 24GB를 초과하면 프로그램이 강제 종료됩니다.

### 4.5 CTS (Cognitive Tree Search) — 우리의 방법

| 항목 | 내용 |
|------|------|
| **정식 명칭** | Cognitive Tree Search |
| **핵심 기술** | DEQ(Deep Equilibrium Model)를 사용하여 **KV 캐시 없이** 트리 탐색 |
| **비유** | MCTS처럼 여러 경로를 탐색하되, **메모리를 거의 쓰지 않으면서** 훨씬 깊이 탐색 가능 |
| **최대 장점** | 깊이 100 이상에서도 VRAM 16.7GB로 **일정** (OOM 없음) |
| **MACs** | 0.65 × 10¹⁴ — 예산의 65%만 사용하고도 최고 성능 달성 |

**CTS가 특별한 이유:**

```
일반 MCTS:  깊이 1 → 깊이 15 → OOM! 💥 (메모리 폭발)
Prefix MCTS: 깊이 1 → 깊이 35 → OOM! 💥
CTS:        깊이 1 → 깊이 50 → 깊이 100+ → 계속 가능! ✅ (메모리 일정)
```

## 5. 결과 수치 읽는 법

### "63.8 ± 0.8" 이란?

| 부분 | 의미 |
|------|------|
| **63.8** | 정답률 63.8% — 500문제 중 약 319문제를 맞힘 |
| **±** | "플러스 마이너스" — 오차 범위를 나타냄 |
| **0.8** | 95% 신뢰구간에서의 오차. 실제 성능은 63.0%~64.6% 사이에 있을 것 |

### 왜 오차 범위가 있는가?

실험을 **5번 반복**(5 seeds)하기 때문입니다. 매번 약간씩 다른 결과가 나오므로,
평균과 오차 범위를 함께 보고합니다. 이것은 "이 결과가 우연이 아니라 신뢰할 수 있다"는 것을 보여줍니다.

- **seed(시드)** = 난수 생성의 시작점. 시드를 바꾸면 같은 실험도 약간 다른 결과가 나옴
- **5 seeds** = 5개의 서로 다른 시드로 5번 실험 → 평균과 표준편차 계산
- **95% CI** = 95% Confidence Interval (신뢰구간) — "100번 반복하면 95번은 이 범위 안에 들어온다"

### "MACs (×10¹⁴)" 란?

| 용어 | 의미 |
|------|------|
| **MACs** | Multiply-Accumulate operations = 곱하기-더하기 연산 횟수 |
| **×10¹⁴** | 10의 14승 = 100조 번 |
| **0.65** | 0.65 × 10¹⁴ = 65조 번의 연산 |

MACs는 "AI가 문제를 풀기 위해 얼마나 많은 계산을 했는가"를 나타냅니다.
**계산을 적게 하면서 정답률이 높을수록** 효율적인 방법입니다.

이 논문에서는 **최대 10¹⁴ MACs** 라는 예산 제한(budget cap)을 설정하고,
그 안에서 누가 가장 좋은 성적을 내는지 비교합니다.

## 11. 자주 묻는 질문(FAQ)

### Q: "Greedy 12.2%"가 논문의 "Greedy 45.2%"와 다른 이유는?

논문의 45.2%는 **최적화된 프롬프팅**(few-shot 예시 제공 등)과 **완전한 학습** 후의 결과입니다.
현재 12.2%는 아직 프롬프팅 최적화가 완료되지 않은 상태의 초기 결과입니다.

### Q: CTS 결과는 왜 안 나오나요?

CTS는 Gemma 모델 위에 **추가 학습**(Stage 1 + Stage 2)을 해야 작동합니다.
현재 학습이 초기 단계(500 steps)이므로 의미 있는 결과가 나오지 않습니다.
그래서 `--skip-cts` 플래그로 건너뛰고 있습니다.

### Q: 왜 Greedy와 Native Think만 실행하나요?

| 전략 | 실행 여부 | 이유 |
|------|:---------:|------|
| Greedy | ✅ 실행 | 기본 성능 측정 — 가장 빠름 |
| Native Think | ✅ 실행 | CTS와 비교할 핵심 대상 |
| SC@14 | ❌ 건너뜀 | 14배 시간 → ~700시간 소요 예상 |
| MCTS | ❌ 건너뜀 | OOM 위험 + 별도 구현 필요 |
| CTS | ❌ 건너뜀 | 학습 미완료 → 의미 있는 결과 불가 |

### Q: OOM이 무엇인가요?

**OOM** = Out Of Memory = 메모리 부족 에러.
GPU VRAM(24GB)을 초과하면 프로그램이 강제 종료됩니다.

### Q: seed(시드)가 무엇인가요?

AI 내부에서 사용하는 **난수(랜덤 숫자)의 시작점**입니다.
같은 시드를 사용하면 같은 결과가 나오고, 다른 시드를 사용하면 약간 다른 결과가 나옵니다.
5개의 시드로 5번 실험하여 결과의 **안정성**을 확인합니다.

### Q: 크래시(crash)로 결과가 유실되었다는 것은?

GSM8K 실험 도중 컴퓨터가 멈추거나(GPU 메모리 과다 사용 등) 프로그램이 비정상 종료되어,
진행 중이던 결과가 저장되지 못한 것입니다. `--resume` 플래그를 사용하면
이미 완료된 MATH-500 결과는 보존하면서 GSM8K부터 다시 시작할 수 있습니다.
