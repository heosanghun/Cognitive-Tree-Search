NeurIPS 2026 (Preprint) 논문 분석
제목: Cognitive Tree Search: Unlocking KV-Cache-Free O(1) Transitions for
      System 2 Inference via Neuromodulated Deep Equilibrium
================================================================================

[Part A] 핵심 내용 요약 (Executive Summary)
--------------------------------------------------------------------------------

1. 문제의식 (Problem)
   - System 2 추론(테스트 타임 탐색, MCTS/ToT 등)은 성능을 크게 올리지만,
     트랜스포머 기반 모델에서는 **각 탐색 경로마다 KV 캐시를 저장**해야 해서
     VRAM 사용량이 트리 크기에 비례해 폭증한다: O(D × W × sequence_length).
   - Gemma 4 E4B(BF16)만으로도 약 16GB를 쓰는 상황에서, 소비자용 GPU(예: RTX 4090 24GB)에서는
     **깊이 15 전후에서 MCTS가 OOM**에 도달할 수 있다는 한계를 지적한다.
   - 4비트 KV 양자화·프리픽스 캐싱 등은 OOM 시점을 늦출 뿐, 근본적인 O(D×W) 복잡도는 해소하지 못한다.

2. 제안 방법 (Cognitive Tree Search, CTS)
   - 명시적 오토회귀 시퀀스 확장 대신, **DEQ(Deep Equilibrium Model)** 로 노드 간 전이를
     **고정점(fixed point) 문제**로 정의: z*_{t+1} = f_{θ,ν}(z*_{t+1}, s_t).
   - Broyden 등 **암시적 솔버**로 고정점을 구하며, 중간 레이어별 KV 캐시 축적 없이
     **전이 시점의 컨텍스트 메모리를 O(1)** 수준으로 제한한다(현재 입력 s_t, 병목 잠복 z*,
     저랭크 야코비 근사 등).
   - 트리 **메타데이터**(Q값, 인덱스, 1D 잠복 벡터 등)는 여전히 깊이에 따라 늘 수 있으나,
     다차원 KV 대비 크기가 매우 작아 실사용 깊이(예: D≤1000)에서 총 VRAM이 사실상 평탄하다고 주장.

3. 아키텍처 (Gemma 4 E4B 적응)
   - Gemma 4의 **Per-Layer Embeddings(PLE)** 를 활용해 사전학습 블록을 **19개 이기종 모듈**
     M = {m1..m19}로 분할·동적 라우팅(경험적으로 Triton 오버헤드와 FLOP 절감의 균형).
   - **RoPE**는 앵커된 부모 컨텍스트 s_t에 **한 번만** 적용해, 희소 라우팅 시 위치 정보 붕괴를 완화.
   - **Soft Thought Bottleneck**: z* ∈ R^{K×d} 형태의 고정 차원 잠복으로 가변 길이 “생각”을 압축 후 LM 헤드로 텍스트 디코딩.

4. 신경조절(Neuromodulator) 메타 정책
   5차원 벡터 ν = [ν_DA, ν_5HT, ν_NE, ν_ACh, ν_Ado]^T 를 경량 메타 정책(LoRA)이 예측하며,
   각 성분이 구체 연산자에 매핑된다:
   - Serotonin (ν_5HT): PUCT 탐색 계수 + 분기 다양성을 위한 초기 잠복에 **등방 가우시안 노이즈 분산** 조절.
   - Acetylcholine (ν_ACh): **희소 라우팅 온도**; Softmax가 뾰족해져 노드당 2~3개 모듈만 활성화해 FLOP·지연 감소.
   - Norepinephrine (ν_NE): Broyden **수렴 허용오차**; 비수렴 시 **가지치기·롤백** 폴백.
   - Dopamine (ν_DA): **단계별 프로세스 보상(TD 오차)** — 희소한 최종 보상만으로는 PPO가 막히는 문제 완화.
   - Adenosine (ν_Ado): 모듈별 MAC을 LUT로 누적한 **계산 비용 누적기**; 예산 초과 시 ACT로 정지.

5. 외부 루프 / 내부 루프
   - 외부: **뉴러모듈레이티드 MCTS** — ν_5HT로 수정된 PUCT, 분기 prior P(s,a)와 결합.
   - 내부: 선택된 가지에서 **DEQ 고정점**으로 z* 갱신 → 최종 z*_final을 LM 헤드에 통과시켜 오토회귀 디코딩.

6. 학습 커리큘럼
   - Stage 1: Gemma 4 E4B 코어 동결, **LoRA 삽입**, OpenMathInstruct 1만 예로 **DEQ/Broyden 워밍업**.
   - Stage 2: 메타 정책을 **PPO**로 학습; Neuro-Critic(LoRA Value Head)은 **z*에 직접 앵커**되어
     어떤 모듈이 스킵되었는지와 무관하게 상태 가치 추정.
   - 총 보상 예: R_total = Σ ν_DA(s_t) + 1{정답} − λ·ν_Ado (논문 식 (6) 요지).
   - VRAM 한도 내 경험 수집: MCTS 시뮬레이션 수/스텝당 제한(예: 3), 옵티마이저 상태 CPU 오프로드 등.

7. 실험 결과 (논문 보고 요약)
   - 단일 RTX 4090, 분기 폭 W=3 고정.
   - VRAM: 표준 MCTS는 깊이 15에서 OOM, CTS는 깊이 100+에서도 약 **16.5~16.7GB** 수준 유지.
   - 지연: CTS 노드당 약 **25ms** 수준(희소 라우팅 덕분), MCTS는 깊이에 따라 증가.
   - Iso-FLOP(약 10^14 FLOP/쿼리) 조건에서 MATH 500, ARC-AGI에서 Native Think·MCTS 대비
     **CTS-Gemma-E4B가 우수**하다고 보고(표 2 수치는 논문 본문 참조).
   - 애블레이션: ν_ACh 없음 → FLOP 3.4×, 지연 >80ms, 정확도 하락; ν_5HT 고정 → MATH 등 하락.

8. 한계 및 향후 연구
   - 마코비안 잠복 압축으로 인한 **정보 병목**(장증명에서 초기 전제 희석).
   - 이산 논리 공간에서 Broyden **진동/비수렴** — Anderson 가속 등 다른 암시적 솔버 검토.
   - 메타데이터 벡터에 대한 **희소 검색 어텐션**으로 먼 조상 회상(밀집 KV 재구축 없이).

9. 부록 구현 디테일 (재현에 중요)
   - 가중치 BF16 로드, Broyden 내부 반복은 **FP32**로 수행 후 수렴 시 BF16 환원(야코비안 역추적 안정화).
   - Broyden: 최대 30회 반복, ν_NE에 따른 동적 tol ∈ [1e-4, 1e-2], 역야코비안 초기값 −I.
   - PPO: lr 3e-5, batch 16, LoRA rank 8 (Q,V), γ=0.95, 1만 스텝, MATH 5k 프롬프트 서브셋 등.
   - 레이어→모듈 매핑: m1–m4(1–8), m5–m8(9–16), m9–m14(17–28), m15–m19(29–42) 레이어(총 42).

Phase 0 — 사전 조건 및 범위 고정
--------------------------------------------------------------------------------
 0-1. 라이선스·배포 정책 확인: Gemma(또는 대체 SLM) 사용 약관, 가중치 배포 경로 확정.
 0-2. 하드웨어 타깃 고정: 최소 1× RTX 4090(24GB) 또는 동등 VRAM; CPU RAM(옵티마이저 오프로드) 확보.
 0-3. 소프트웨어 스택: Python 3.10+, PyTorch 2.x(CUDA), Flash-Attn(가능 시), Triton, transformers/가속 백엔드.
 0-4. 논문 수식·표·부록을 **요구사항 체크리스트**(O(1) 메모리 주장, W=3, Iso-FLOP 정의 등)로 문서화.

--------------------------------------------------------------------------------
Phase 1 — 베이스 모델 로딩 및 19모듈 매핑
--------------------------------------------------------------------------------
 1-1. Gemma 4 E4B(또는 동등 PLE 구조) 로더 구현: BF16 로딩, 메모리 프로파일링 훅.
 1-2. Table 3에 따라 **레이어 인덱스 → 모듈 인덱스** 정적 테이블 `layer_to_module[i]` 구현.
 1-3. 각 모듈 `m_j`를 “해당 레이어 블록의 순전파 callable”로 래핑; 입력은 (z, context_embeds) 형태로
      통일할 인터페이스 설계(후속 DEQ와 호환).
 1-4. RoPE: 논문대로 **앵커 컨텍스트 s_t에만 1회 적용**하는 경로와 단위 테스트(스킵 시 깨짐 방지).

--------------------------------------------------------------------------------
Phase 2 — Universal Latent Space & Soft Thought Bottleneck
--------------------------------------------------------------------------------
 2-1. z ∈ R^{K×d} 텐서 표현, 차원 K,d를 설정 가능하게(논문 기본값 또는 튜닝).
 2-2. 부모 노드 텍스트 s_t → 토크나이저 → 임베딩 → 앵커 표현 파이프라인.
 2-3. 모듈 입력 결합 규칙: 각 m_i(z, s_t)가 DEQ 잔차맵 `f`의 일부가 되도록 합성 함수 정의.
 2-4. LM 헤드로부터의 **출력 텍스트 디코딩** 경로: z*_final → (필요 시 짧은 AR) → st+1 텍스트 확정.

--------------------------------------------------------------------------------
Phase 3 — DEQ 고정점 솔버 (Broyden + IFT 역전파)
--------------------------------------------------------------------------------
 3-1. 고정점 방정식 z* = f_{θ,ν}(z*, s_t)에 대한 순전파 Broyden 구현:
      - FP32 내부 루프, max_iter=30, tol=ν_NE 스케줄, 역야코비안 초기 −I.
      - 수렴 실패 시 **가지 제거·상위 롤백** API.
 3-2. **암시적 미분**: 고정점에서 IFT 기반 backward (라이브러리: torchdeq 등 검토 또는 자체 구현).
 3-3. 단위 테스트: 작은 차원에서 수치적 기울기와 비교, NaN/발산 모니터링.
 3-4. 메모리 프로파일: 노드 전이 1회당 할당이 KV 캐시 누적 없이 일정한지 검증.

--------------------------------------------------------------------------------
Phase 4 — Acetylcholine 희소 라우팅 & Triton 커널
--------------------------------------------------------------------------------
 4-1. 라우팅 가중: α_i = softmax( (W_g z) / ν_ACh )_i — ν_ACh 스칼라 또는 벡터 확장 설계.
 4-2. 희소 실행: 상위 2~3개 모듈만 선택하는 **top-k 라우팅**과 논문 Softmax 온도 방식 정합.
 4-3. PyTorch 기준 **기능 동등(reference)** 구현 후, Ada 타깃 **Triton fused kernel**로 가속(부록 A.2).
 4-4. 벤치마크: 노드당 지연 목표(논문 ~25ms) 대비 프로파일링.

--------------------------------------------------------------------------------
Phase 5 — 뉴러모듈레이터 메타 정책 네트워크
--------------------------------------------------------------------------------
 5-1. 상태 인코딩: s_t(및 선택적 트리 통계) → 소형 네트워크 입력.
 5-2. 출력: ν ∈ R^5, 분기 prior P(s,a) (폭 W).
 5-3. LoRA: rank r=8, 적용 위치(Q,V) 논문과 동일하게; 파라미터 카운트·체크포인트 포맷 정의.
 5-4. PUCT 수정: U(s,a) = ν_5HT · P(s,a) · sqrt(N(s)) / (1+N(s,a)) (식 (4) 구현).
 5-5. 초기 잠복 z0에 **등방 가우시안 노이즈**(ν_5HT로 스케일) 삽입.

--------------------------------------------------------------------------------
Phase 6 — Adenosine LUT & ACT (계산 예산)
--------------------------------------------------------------------------------
 6-1. 모듈별 MAC/FLOP 사전 계산 → LUT 구축(정적 JSON/테이블).
 6-2. 라우팅 결과에 따라 ν_Ado 누적; 임계 τ_budget 초과 시 전이 중단·현재 z로 디코딩.
 6-3. Iso-FLOP 실험과 연동: 쿼리당 총 FLOP 상한 추적 도구.

--------------------------------------------------------------------------------
Phase 7 — 외부 MCTS 루프
--------------------------------------------------------------------------------
 7-1. 표준 MCTS 데이터 구조(노드, 엣지, N, Q, prior) 구현.
 7-2. 확장 시: W개 가지 생성 = ν_5HT 노이즈 + DEQ 전이 W회(또는 배치화).
 7-3. 시뮬레이션 횟 제한(논문: 스텝당 3 등)으로 VRAM 내 replay/버퍼 유지.
 7-4. 백업 규칙, 불안정 가지(솔버 실패) 처리.

--------------------------------------------------------------------------------
Phase 8 — Neuro-Critic & 보상 설계
--------------------------------------------------------------------------------
 8-1. Value head를 **z*** 에 직접 연결(모듈 스킵 불변 평가).
 8-2. TD 타깃으로 ν_DA(프로세스 보상) 계산; 터미널 정답 보상, ν_Ado 패널티 포함(식 (6)).
 8-3. PPO: GAE, 클립, 배치 16, lr 3e-5, γ=0.95.

--------------------------------------------------------------------------------
Phase 9 — 학습 Stage 1 (DEQ 워밍업)
--------------------------------------------------------------------------------
 9-1. OpenMathInstruct 1만 샘플 서브셋 로더.
 9-2. 고정점 수렴률·손실 로깅; 실패율이 높으면 tol/초기값/스케일 튜닝.

Phase 10 — 학습 Stage 2 (PPO + MCTS 경험 수집)

10-2. 옵티마이저 상태 CPU 오프로드 옵션.
10-3. 체크포인트: 메타 정책·LoRA·솔버 하이퍼파라미터 동시 저장.
